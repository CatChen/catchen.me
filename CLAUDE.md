# CLAUDE.md

## Project Overview

Personal homepage at [catchen.me](https://catchen.me). Static site generated by [Harp](https://harp.sh/), deployed on Netlify.

## Commands

```bash
yarn install          # install dependencies
yarn start            # dev server at localhost:9000
yarn build            # compile to www/
```

**After any code change, run `yarn build` to validate.** This is the only automated check — there are no tests. A successful build confirms the templates compile and the CSS processes without errors.

## Localization System

The site supports `en` and `zh` locales. The URL structure is `/{locale}/{page}`.

**Data hierarchy:**

- `public/_data.json` — base data for all pages (written in English)
- `public/zh/_data.json` — Chinese overrides (merged on top of base data)
- `public/en/_data.json` — English overrides (currently `{}`, base data serves as English)

Each locale page delegates to `_partials/any.jade`, which deep-merges the base and locale data, then dispatches to the matching partial in `public/_partials/`.

**Key Harp template variables in `any.jade`:**

- `current.source` — the page name (`index`, `resume`, `services`, `projects`, `contents`)
- `current.path[0]` — the locale (`en` or `zh`)

**Locale metadata** (nav labels, 404 text, date formats) lives in `public/_data.json` under `locales.en` and `locales.zh`.

**Adding/editing content:**

- Edit `public/_data.json` for English content (base data)
- Edit `public/zh/_data.json` for Chinese equivalents — the zh file only needs to include keys that differ from English

**Data merge behavior:** `deepCopy` deep-merges objects; array entries from `zh/_data.json` replace by index (they don't append). The zh resume only carries partial data per experience — it merges on top of the base array's structure.

## CSS

Bootstrap is built from SCSS source to allow excluding unused modules.

- `public/css/bootstrap-custom.scss` — selectively imports Bootstrap SCSS modules; add imports here to enable more components
- `public/css/bootstrap-with-override.scss` — imports `bootstrap-custom`, then adds project-specific overrides

Dark mode uses `$color-mode-type: media-query` (respects OS preference via `prefers-color-scheme`).

## JavaScript

Bootstrap JS is copied from `node_modules/bootstrap/dist/js/` into `public/js/` by `yarn copy-js` (runs automatically before build and dev server). `public/js/` is gitignored.
